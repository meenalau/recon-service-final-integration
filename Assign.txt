What business problem is this solving?

Currently your flow is:

Trade Created
      |
      V
Kafka Event
      |
      V
BREAK DETECTED

Suppose a reconciliation break exists:

Break Id = BRK001
Status = OPEN

An operations user resolves it:

PUT /api/v1/recon/breaks/BRK001/resolve

Without Kafka:

REST API
   |
   V
Database Updated

Only the current application knows the break was resolved.

Why Kafka?

Other systems may need to know:

Audit Service
Notification Service
Reporting Service
Dashboard Service

Instead of calling all these directly:

Recon Service
   |
   +--> Audit
   +--> Reporting
   +--> Dashboard

publish an event:

Recon Service
      |
      V
Kafka Topic
recon.breaks.resolved
Any interested system can subscribe.
